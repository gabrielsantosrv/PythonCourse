git clone <link do repositório> - baixar o repositório pela 1a vez

git status - mostra as alterações realizadas no projeto

git diff <arquivos> - indica as alterações feitas num arquivo

git add <arquivos> - adiciona os arquivos que serão versionados

git commit -m "<mensagem>" - cria uma versão do seu código

git push - publica as suas alterações (sempre faça um git pull antes de um git push)

git pull - baixa as alterações do diretório remoto (baixa do github/gitlab, etc)
